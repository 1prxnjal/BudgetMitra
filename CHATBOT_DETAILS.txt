## Section C: Context & Data Handling
Q7. Contextual Memory Usage: [x] Session-based memory (Last 10 conversation turns)
Q8. Flow of Data in the Chatbot:
1. UI (React): User types a message or clicks a dynamic prompt.
2. Frontend: Axios/Fetch sends the message + Budget Context (Income/Expenses/Loans) to the backend.
3. Backend (Flask/FastAPI): Receives the payload, formats the financial data into a system prompt, and appends chat history.
4. AI Model (Groq): Receives the rich prompt and generates a streaming response using LPU™ technology.
5. Response: The backend streams the text back to the UI.
6. UI: ReactMarkdown renders the text into professional headers and tables for the user.

## Section F: Implementation Evidence (Code Snippet)
Q12. API Call Snippet:

```python
# Backend API Call to Groq (app.py)
completion = client.chat.completions.create(
    model="llama-3.3-70b-versatile",
    messages=[
        {"role": "system", "content": system_instruction}, # Personalized Finance Context
        {"role": "user", "content": user_message}
    ],
    temperature=0.3,
    stream=True
)

for chunk in completion:
    content = chunk.choices[0].delta.content
    if content:
        yield content # Streams back to React UI
```
